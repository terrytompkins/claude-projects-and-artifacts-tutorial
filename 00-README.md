# Promptly-at-9: Claude Projects + Artifacts Demo Package

This package contains the prompts, presentation outline, and presenter runbook for a 30-minute Promptly-at-9 session titled:

> **Beyond the Chat: Building Reusable Workspaces and Apps with Claude**

## Package contents

| File | Purpose |
|---|---|
| `01-project-workspace-prompt.md` | Generates the complete Pet Care Coach knowledge bundle for a Claude Project |
| `02-artifact-generation-prompt.md` | Generates the initial Pet Care Coach Journey Explorer Artifact |
| `03-live-demo-prompts.md` | Contains the live Project prompts, live Artifact enhancement prompt, and recovery prompt |
| `04-presentation-outline.md` | Defines the four-slide presentation and 30-minute flow |
| `05-presenter-runbook.md` | Provides preparation steps, speaking notes, demo sequence, timing, and fallback options |

## Recommended preparation order

1. Run Part 1, then Part 2, of the prompt in `01-project-workspace-prompt.md` in the same Claude chat.
2. Create a Claude Project named **Pet Care Coach Product Workspace**.
3. Copy the generated project instructions into the Project Instructions field.
4. Upload the remaining generated files as Project Knowledge.
5. Run the prompt in `02-artifact-generation-prompt.md` inside the Project.
6. Work through every item in the Artifact's functional-verification checklist. If anything fails, use the emergency repair prompt in `03-live-demo-prompts.md` to fix it before moving on.
7. Build the four slides described in `04-presentation-outline.md`.
8. Rehearse using `05-presenter-runbook.md`.
9. Keep `03-live-demo-prompts.md` open during the session.

## A note on file numbering

This package's own files are dash-numbered (`01-project-workspace-prompt.md`, `02-artifact-generation-prompt.md`, ...). The files *generated by* `01-project-workspace-prompt.md` and uploaded to the Project are underscore-numbered (`00_PROJECT_INSTRUCTIONS.md`, `01_PRODUCT_OVERVIEW.md`, ...). These are two separate numbering schemes — "file 01" means something different depending on which set you're referring to.

## Recommended live-demo strategy

Use a pre-created Project and a pre-created Artifact. During the class:

- Show the Project structure.
- Run one or two short Project-grounded prompts.
- Open the completed Artifact.
- Demonstrate its existing interactions.
- Make one controlled live enhancement.
- Use the emergency repair prompt only if needed.

This keeps the session reliable while still showing live generative work.

## Session objective

By the end of the demonstration, the audience should understand:

- How a Claude Project differs from a normal chat.
- How Project Instructions and Project Knowledge serve different purposes.
- How multiple chats can reuse the same Project context.
- How an Artifact can turn Project knowledge into an interactive experience.
- How the same knowledge can support product, user-experience, architecture, and QA workflows.
